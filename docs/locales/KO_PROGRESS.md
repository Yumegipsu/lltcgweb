# Korean (ko) localization progress

`document.documentElement.lang = 'ko'`; `body.locale-ko` applies `Noto Sans KR` (falls back to `Nunito`).

Unlike **es** (which keeps `name_en` for characters/songs), Korean uses **phonetic/semantic Korean names** for characters and song titles, resolved at render time via `KO_NAME_MAP` / `KO_SONG_MAP` (see [Naming policy](#naming-policy)). Card rules use **`text_ko`** with Korean skill brackets.

## Naming policy

Korean **diverges from Spanish** on this point:

- **Spanish (`es`)**: character and song names stay **English** (`name_en`); only UI strings and (eventually) card rules text are translated.
- **Korean (`ko`)**: character and song names are shown in **Korean transliteration** (`KO_NAME_MAP` / `KO_SONG_MAP`), using either:
  - `phonetic` — direct sound transliteration (e.g. `Snow halation` → 스노우 할레이션), used for most English/katakana-styled titles.
  - `semantic` — meaning-based translation (e.g. `Aozora Jumping Heart` → 파란 하늘 점핑 하트), used for Japanese romaji titles with translatable meaning.
  - `sino` — Sino-Korean reading of kanji-derived terms (e.g. `Kokon Touzai` → 고금동서), used for yojijukugo/kanji-compound titles.
- Group names (`groups` in `ko_names.json`) are **partially localized**: idol-group names that are already Latin-script brand marks (`μ's`, `Aqours`, `Superstar`, `Liella!`, `A-RISE`) stay as-is; Japanese-origin unit names (`Nijigasaki`, `Hasunosora`) get Korean readings. Sub-unit names (`subunits`) mostly stay Latin/English except a couple of Japanese-titled ones (`みらくらぱーく!` → 미라클파크!).
- **Card rules text** (`text_ko`, not yet written) should quote character/song names **in the same Korean form** used by `KO_NAME_MAP`/`KO_SONG_MAP` for consistency with the card list UI, once M5 starts — this differs from the ES policy of quoting English names in Spanish rules text. Confirm with the localization owner before starting M5 in case this policy should instead match ES (quoted names untranslated).

## M4 — game log + prompt text localization (`log_i18n.js`)

Mirrors the `es` rule tables (`STRUCTURAL_PHRASE_RULES_ES` / `PHRASE_RULES_ES` / `EFFECT_RULES_ES` / `translateStructuredLineEs`) with a Korean counterpart, wired into the same locale dispatch as `ja`/`es`:

- `STRUCTURAL_PHRASE_RULES_KO`, `PHRASE_RULES_KO`, `EFFECT_RULES_KO`, `translateStructuredLineKo` — high-frequency turn/phase/Live log lines and effect suffixes.
- `SKILL_BRACKETS_KO` — `replaceSkillBrackets(msg, map)` now takes an optional map (was JA-only); KO prompt/log text runs `[On Enter]` → `[등장 시]`, `[Live Start]` → `[라이브 개시]`, `[Activated]` → `[기동]`, etc.
- `replaceCardNamesKo` / `buildNamePairsKo` — unlike JA (which swaps English → Japanese via the card catalog's own `name`/`name_en` pair), KO resolves English → Korean via `LLTCG_I18N.cardLocaleName()` (`KO_NAME_MAP` / `KO_SONG_MAP`); cards without a Korean name mapping are left in English (no-op), same as the card-list UI.
- `localizePromptTextKo` / `localizeLogMessageKo` — dispatched from `localizePromptText` / `localizeLogMessage` when `getLocale() === 'ko'`.
- `EXACT_LOG_KEYS` (`log.dividerTurnBegin`, `log.disconnectedWin`, `log.hasNoValidLive`, `log.chooseSuccessLive`, `log.scoreTiedBlocked`, `log.scoreTiedCap`, `log.cpuDeck`) resolve through `STRINGS.ko.log.*` (already present in `locales/ko.json` from M0).
- Like `es`, KO does **not** call `translateOpponentLabels` (CPU/difficulty labels stay in English) — only `ja` has that pass.
- `index.html` `RULE_KW_BRACKET_LABELS_KO` / `RULE_KW_KO_TO_EN` added alongside the `ES` ones so card-rules-text bracket keywords (`[On Enter]`, `[Activated]`, …) get Korean tooltip titles/bodies from `STRINGS.ko.skillKw` even while `card.text_ko` doesn't exist yet (falls back to English body with Korean bracket label). `cardRulesDisplayText()` also has a `ko` branch (`card.text_ko || en`) ready for M5.
